Our unique selling point is that not only are we checking the validity of the EDI file, we are checking the integrity of the data itself.

It doesn't matter if the EDI file is structured correctly if the data it contains doesn't make contextual sense.

We can use our own datasets, as well as third-party or imported datasets, to cross-reference data and ensure there are no data mismatches. For example if an Amazon global location number was being used against the wrong address given, we can catch that for you.

We also have the ability to verify a wide range of datatypes such as the GS1 serial shipping container code or ISO vehicle identity numbers.

Quickly generate useful outputs and reports such as a bayplan view built from a BAPLIE message.

The list of outputs is growing everyday as more support is implemented for many different EDI agencies.
