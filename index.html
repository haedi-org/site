✨ Validate

Not only do we have the ability to verify the structure of an EDI file, we can check the integrity of the data itself.

This includes third-party specifications, ISO standards, more contextual data such as address codes (i.e. postal codes, Amazon GLN, etc.)

Being able to verify the integrity of a GS1 container code, or ISO VIN, can help catch data mismatch before it reaches the recipient.

✨ Convert

Quickly convert your EDI files to a more accessible format, such as XML or JSON, to make data ingestion easier. This can include our cross-referenced definitions so you can easily interrogate each data point.

$ haedi ./my_baplie.edi --xml > ./my_baplie.xml
<LOC>
  <LOC01><!--3227: Place/location qualifier-->83<!--Place of delivery (by on carriage)--></LOC01>
  <LOC02>
    <LOC0201><!--3225: Place/location identification-->AUSYD<!--WHITE BAY--></LOC0201>
    <LOC0202><!--1131: Code list qualifier-->139<!--Port--></LOC0202>
    <LOC0203><!--3055: Code list responsible agency, coded-->6<!--UN/ECE--></LOC0203>
  </LOC02>
</LOC>

✨ API

There are many EDI agencies, and their specifications change all the time. By use of our API, you can query the exact data you need without being flooded with updates and the need to store terabytes of data that you only need a few kilobytes of.

If there are any agencies that we do not have categorized, let us know and we'll do our best to get them integrated.

haedi is a data preservation project too - we believe these communication standards shouldn't be lost to time, they should be archived.
